# Architecture

Two-file split: `ui.py` is pure presentation (CustomTkinter widget tree, colors, layout, asset loading), `main.py` is all application logic wired onto the `OmniFlowUI` instance.

- **`ui.py`** defines `OmniFlowUI(ctk.CTk)`. It builds the entire widget tree in `init_ui()` — a single scrollable column: header → URL input box → an instructional text screen → a "checking" screen (spinner/status/cancel) → a result stack (thumbnail+metadata box, quality-selector box, download-progress/success box). Widgets are exposed as `self.*` attributes (e.g. `self.entry_link`, `self.btn_download`, `self.seg_quality`, `self.progressbar`) for `main.py` to read/configure directly — there is no callback/event abstraction, `main.py` pokes widget state straight through `app.<widget>.configure(...)`.
- **`main.py`** creates `app = OmniFlowUI()` at module scope, defines all business logic as free functions, then at the bottom wires them onto the UI (`app.btn_download.configure(command=download_logic)`, `app.link_var.trace_add(...)`, etc.) before calling `app.mainloop()`.
- **Threading model**: every `yt-dlp`/network call (`get_video_info`'s `fetch()`, `download_logic`'s `run()`) runs in a daemon `threading.Thread`; results are marshaled back to the Tk main thread via `app.after(0, ...)`. A module-level `current_proc` (subprocess handle) and `stop_event` flag implement cancellation — `cancel_process()` sets `stop_event` and terminates `current_proc`; long-running loops check `stop_event` between reads.
- **Video info fetch vs. download are two separate `yt-dlp` invocations**: `get_video_info()` runs `yt-dlp --simulate --dump-json` to populate the thumbnail/title/available-resolutions UI, then `download_logic()` runs a second `yt-dlp` invocation with format selectors built from the chosen quality (`-f bestvideo[height<=H]+bestaudio/...`, or `-x --audio-format mp3` for audio-only) and parses stdout progress lines (`\d+(\.\d+)?%`) to drive the progress bar.
- **Persisted state**: `config.json` at the repo root stores the last-used download folder and notification toggle (`load_session`/`save_session` in `main.py`). This file is tracked in git and contains a machine-specific path — expect it to reflect whoever last ran the app locally, not a template.
- **Platform detection**: `get_platform_info()` in `main.py` does simple substring matching on the URL to pick a label/color/tag icon (YouTube/Instagram/TikTok/Facebook/RedNote); anything else falls through to a generic "Link" tag.
- **`Assets/`** holds SVG icons (`Assets/Icons`), the app logo (`Assets/Logo`), and per-platform tag icons (`Assets/Tags`), all rasterized on load via `cairosvg` in `ui.py`.
